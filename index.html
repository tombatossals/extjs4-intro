Qué es ExtJS

Framework de desarrollo web de aplicaciones cliente que nos proporciona facilidades para comunicarnos con el servidor mediante un interfaz REST y varios componentes que acercan la apariencia de aplicación de escritorio a la aplicación web que desarrollamos.


Qué añade ExtJS 4 a ExtJS 3

  • Más rápido.
  • Más estable.
  • Más fácil de aprender y usar.

La carga de librerías y dependencias se realiza de manera dinámica dentro de nuestra aplicación.

Estructura de los archivos de desarrollo

Aprovechando el patrón de desarrollo MVC (que luego comentaremos), existe un convenio de estructuración del código de un proyecto ExtJS 4 en archivos y directorios.

En ExtJS 3

Los archivos .js los poníamos todos en la misma carpeta sin orden ni concierto.


En ExtJS 4

Utilizamos una estructura separando en carpetas los modelos, stores, vistas y controladores.

El patrón MVC

Esta versión de ExtJS nos permite (no obliga) a utilizar un patrón de desarrollo Modelo/Vista/Controlador en nuestro código, con algunas características y peculiaridades.


En ExtJS 3

Cada componente que desarrollamos es un módulo independiente que lo hace todo por sí mismo: recibe/emite eventos, dibuja el interfaz de usuario y define sus Stores todo en un único archivo.

Ejemplo: http://devel.uji.es/ocw/js/cursoForm.js

En ExtJS 4

Separamos los datos, el acceso a los datos, la interfaz de usuario y la lógica en archivos independientes.

  • Vista: Se encarga de representar en pantalla la interfaz de usuario.
  • Controlador: Se encarga de la lógica. Escuchará los eventos que se producen en la vista y reaccionará con las acciones adecuadas.
  • Modelo: Se encarga de definir la estructura de la información para hacer persistir los datos.
  • Store: Se encarga de comunicar un modelo con el backend.

Selectores

Una de las novedades que hemos encontrado más útiles. Nos proporcionan la manera de buscar y acceder cualquier componente de nuestra aplicación de manera directa.


up() / down()

A partir de un componente podemos bajar o subir en el arbol DOM hasta encontrar la siguiente ocurrencia.


Ejemplo:


function pintaBotondeRojo(formulario) {
    var boton = formulario.down("button[action=cancel]");
    boton.setStyle("background", "red");
}

Selectores

No siempre tenemos una referencia a un componente y hay que buscarlo. En ExtJS3 nos guardábamos referencias dentro de los componentes. En ExtJS4 tenemos el ComponentQuery.


Ejemplo:


function pintaBotondeRojo() {
    var formulario = Ext.ComponentQuery.query("form[name=principal]")[0];
    var boton = formulario.down("button[action=cancel]");
    boton.setStyle("background", "red");
}
